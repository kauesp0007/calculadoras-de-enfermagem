Passo 1: Como criar o arquivo do artigo (O Início)
Toda a mágica começa na criação do documento base onde você escreverá o texto.

Abra o seu Visual Studio Code.

No painel lateral esquerdo (Explorador), localize e abra a pasta chamada posts-markdown.

Clique com o botão direito em cima da pasta posts-markdown e selecione "New File" (Novo Arquivo).

Nomeie o arquivo seguindo a regra rigorosa de URL amigável:

Use apenas letras minúsculas.

Não use espaços, acentos ou caracteres especiais (ç, ~, ^).

Separe as palavras apenas com hífens (-).

O arquivo deve terminar obrigatoriamente com a extensão .md.

Exemplo correto: cuidados-com-sonda-vesical.md

Nota de SEO: O nome exato que você der a este arquivo .md será o link oficial da sua página na internet.

⚙️ Passo 2: O Cabeçalho Obrigatório (Front Matter)
Todo arquivo .md deve iniciar com um bloco de configurações. É através dele que o sistema lê os dados vitais do artigo e organiza o índice e o SEO.

Copie exatamente o bloco de código abaixo e cole na primeira linha do seu arquivo:

Markdown
---
title: Escreva Aqui o Título do Seu Artigo
description: Escreva aqui um resumo curto (1 ou 2 frases) sobre o que o artigo ensina. Isso aparecerá no Google e no índice do blog.
date: 29 de Maio de 2026
keywords: enfermagem, uti, termo1, termo2, termo3
image: https://www.calculadorasdeenfermagem.com.br/img/nome-da-sua-imagem-de-capa.webp
---
O que cada linha faz:

title: O título principal que vai para as tags <meta> do HTML e para a aba do navegador.

description: O resumo que aparece nos resultados de pesquisa do Google e abaixo do título na página inicial do blog.

date: A data que ordena o artigo no painel geral, jogando os mais recentes para o topo.

image: A miniatura do seu artigo. Ela aparece no índice do blog e atua como a "capa invisível" lida pelo WhatsApp, Facebook e Google Discover quando o link é compartilhado.

✍️ Passo 3: Escrevendo o Conteúdo (Formatação Markdown)
Todo o texto do artigo deve ser escrito abaixo do último --- do cabeçalho. Você não usará botões para formatar o texto, mas sim símbolos simples que o automatizador converterá perfeitamente.

Títulos e Subtítulos
Use a hashtag (#) seguida de um espaço em branco:

Markdown
# Introdução (Título Principal - H1, fica bem grande e azul escuro)

## Fisiopatologia (Subtítulo - H2, possui uma linha divisória embaixo)

### Sinais e Sintomas (Tópico Menor - H3, cor azul clara)
Parágrafos
Para mudar de parágrafo, você deve pular uma linha em branco.

Markdown
Este é o primeiro parágrafo do texto.

Este é o segundo parágrafo.
Destaques (Negrito e Itálico)
Envolva as palavras com asteriscos:

Markdown
Este texto está em **negrito** e este está em *itálico*.
Listas (Tópicos com Bolinhas)
Inicie a linha com um asterisco (*) e um espaço:

Markdown
* Primeiro item da lista.
* Segundo item da lista.
* Terceiro item da lista.
Inserir Imagens no Corpo do Texto
O script automaticamente deixará a imagem responsiva, com cantos arredondados e sombra (w-full rounded-xl shadow-lg my-10):

Markdown
![Descrição textual para deficientes visuais e SEO Google](/img/nome-da-imagem.webp)
Links para Outras Páginas
O texto clicável vai entre colchetes [] e a URL de destino vai entre parênteses ():

Markdown
Acesse a nossa [Calculadora de Gasometria](/gasometria.html) para saber mais.
🚀 Passo 4: Transformar o .md em HTML e Publicar
Com o texto escrito e formatado, chegou a hora de gerar a página real e colocá-la no ar.

Salve o arquivo: Pressione Ctrl + S no Visual Studio Code para salvar o seu .md.

Abra o Terminal: No próprio VS Code, abra o terminal integrado.

Execute o construtor: Digite o comando abaixo e aperte Enter:

Bash
npm run blog
Verificação Automática: O seu automatizador irá gerar a página HTML completa dentro da pasta blog e atualizará automaticamente o arquivo blog/index.html para incluir o seu novo artigo na lista geral.

Envie para o Servidor (GitHub Desktop):

Abra o GitHub Desktop.

Ele identificará que novos arquivos foram criados e modificados.

Escreva o nome da atualização no campo de Summary (Ex: "Publicação do artigo Sonda Vesical").

Clique em Commit to main.

Clique em Push origin para enviar para a nuvem.

Assim que os servidores processarem a atualização, a página do seu blog estará ao vivo com URLs amigáveis, tags <meta> implementadas e um layout 100% responsivo.
