Parte 1: Como criar o ficheiro .md (O Início)
Abra o seu VS Code.

No painel à esquerda, abra a pasta posts-markdown.

Clique com o botão direito na pasta e selecione "New File" (Novo Ficheiro).

Nomeie o ficheiro: Use letras minúsculas e hífens em vez de espaços. O nome do ficheiro será o link da página.

Exemplo: cuidados-com-sonda-vesical.md

Exemplo: interpretacao-gasometria-completa.md

Parte 2: O Cabeçalho Obrigatório (Front Matter)
Todo ficheiro .md DEVE começar com este bloco. É ele que diz ao motor do blog qual é o título, a data e a imagem de capa para o Google (SEO).

Copie e cole isto na primeira linha do ficheiro e preencha os dados:

Markdown
---
title: Escreva Aqui o Título do Seu Artigo
description: Escreva aqui um resumo curto (1 ou 2 frases) sobre o que o artigo ensina. Isso aparecerá no Google e no índice do blog.
date: 20 de Fevereiro de 2026
keywords: enfermagem, uti, termo1, termo2, termo3
image: https://www.calculadorasdeenfermagem.com.br/img/nome-da-sua-imagem-de-capa.webp
---
Explicação dos Campos:

image: Esta é a imagem "Thumbnail". Ela aparece quando você partilha o link no WhatsApp/Facebook e serve para o Google Discover. Ela NÃO aparece no topo do texto (que é azul), ela é interna para SEO.

Parte 3: Escrevendo o Conteúdo (Comandos Sequenciais)
Logo abaixo do último --- do cabeçalho, você começa a escrever o texto. O Markdown usa símbolos simples para formatar.

Aqui estão os códigos que você deve usar, na ordem que preferir:

1. Títulos e Subtítulos
Use hashtag e espaço antes do texto.

Markdown
# Introdução (Título Principal - H1)
Escreva sua introdução aqui...

## Fisiopatologia (Subtítulo - H2)
Texto sobre fisiopatologia...

### Sinais e Sintomas (Tópico Menor - H3)
Texto sobre sinais...
2. Parágrafos
Apenas escreva normalmente. Para mudar de parágrafo, pule uma linha.

Markdown
Este é o primeiro parágrafo.

Este é o segundo parágrafo (pulei uma linha acima).
3. Negrito e Itálico
Markdown
Isso é um texto em **negrito** para destacar.
Isso é um texto em *itálico* para ênfase leve.
4. Listas (Tópicos)
Use asterisco * ou hífen - seguido de espaço.

Markdown
* Lavar as mãos.
* Preparar o material.
* Explicar o procedimento.
5. Inserir Imagens no Corpo do Texto
É aqui que as imagens aparecem grandes e bonitas para o leitor, como na página "Teorias de Enfermagem".

Markdown
![Descrição da imagem para cegos](/img/nome-da-imagem.webp)
Nota: A imagem deve estar na sua pasta /img ou /public/img.

6. Links para outros sites ou páginas
Markdown
Clique aqui para ver a [Calculadora de Gasometria](/gasometria.html).

Parte 4: Como transformar o .md em HTML
Depois de salvar o seu ficheiro .md:

Abra o terminal do VS Code.

Digite o comando que configurámos:

Bash
npm run blog
O que acontece: O script lê o seu .md, junta com o post.template.html, processa as imagens e o layout, e cria um ficheiro novo na pasta blog/ (ex: blog/puncao-venosa.html).

Ao mesmo tempo, ele atualiza o blog/index.html adicionando este novo artigo à lista.

Parte 5: Perguntas e Respostas Frequentes
1. A imagem do cabeçalho (image: ...) aparece na página inicial do blog?
No código atual que configurámos para o índice (index.html), aparecem apenas: A Data, o Título e a Descrição.

Motivo: Para manter o layout "limpo" e focado em leitura rápida, similar a uma lista técnica.

Onde a imagem aparece então? Ela fica "invisível" no código da página (metatags). Quando alguém colar o link do seu blog no WhatsApp ou Facebook, essa imagem aparecerá como miniatura no card de pré-visualização.

2. Os usuários do Google podem achar o artigo direto?
Sim, absolutamente.
Cada postagem vira uma página HTML independente (ex: .../blog/puncao-venosa.html).

O Google indexa essa página individualmente.

Se alguém pesquisar "Como fazer punção venosa enfermagem", o Google pode mostrar o link direto para essa página. O usuário não precisa passar pelo índice (index.html). Ele cairá direto no conteúdo.

3. Como o Google descobre a página nova?
O Google descobre através de links internos.

O seu menu-global leva ao blog/index.html.

O blog/index.html tem links para todos os artigos.

O robô do Google entra na home, vai para o blog, e do blog vai para o artigo.

(Opcional avançado: você pode adicionar ao sitemap.xml, mas os links internos já funcionam bem).
