✅ PASSO A PASSO — USO NO DIA A DIA
🟢 PASSO 1 — ADICIONAR UM ARQUIVO NOVO

Você pode adicionar:

✔ Foto → img/
✔ PDF / Documento → docs/
✔ Vídeo → videos/

📌 O nome do arquivo pode ter espaço, hífen ou underline
Exemplos válidos:

Escala de Braden.webp
escala-braden.webp
escala_braden.webp


👉 O sistema corrige automaticamente o título.

🟢 PASSO 2 — ATUALIZAR A biblioteca.json (AUTOMÁTICO)

Depois de adicionar arquivos nas pastas, execute:

npm run biblioteca

🔧 O que esse comando faz?

✔ Escaneia img/, docs/, videos/
✔ Detecta arquivos novos
✔ Cria entradas automáticas no biblioteca.json
✔ Não duplica arquivos já existentes

📌 Você NÃO edita biblioteca.json manualmente
(o sistema faz isso por você)

🟢 PASSO 3 — GERAR AS PÁGINAS HTML DA BIBLIOTECA

Agora transforme cada item da biblioteca em uma página HTML própria:

node build-biblioteca.js

🔧 O que esse comando faz?

✔ Lê biblioteca.json
✔ Usa item.template.html
✔ Gera arquivos como:

/biblioteca/escala-de-braden.html
/biblioteca/protocolo-rcp-adulto-2020.html


✔ Cada item vira uma página indexável no Google
✔ SEO automático (title, description, canonical, OG)

🟢 PASSO 4 — ATUALIZAR A PÁGINA DE DOWNLOADS

Sempre que a biblioteca muda, execute:

node build.js

🔧 O que esse comando faz?

✔ Atualiza downloads.html
✔ Cria /downloads/page2.html, /page3.html etc
✔ Atualiza paginação
✔ Atualiza cards com links para a biblioteca

📌 Nunca edite downloads.html manualmente
Sempre edite:

downloads.template.html

🟢 PASSO 5 — ATUALIZAR O SITEMAP (SEMPRE POR ÚLTIMO)

Depois que tudo foi gerado:

node generate-sitemap.js

🔧 O que esse comando faz?

✔ Detecta:

downloads.html

/downloads/pageX.html

/biblioteca/*.html
✔ Atualiza sitemap.xml
✔ Ajuda o Google a indexar tudo

🟢 PASSO 6 — SUBIR PARA O GITHUB

Depois de rodar todos os comandos:

git status
git add .
git commit -m "Atualização automática da biblioteca"
git push


✔ GitHub Pages atualiza o site
✔ Google rastreia novas páginas via sitemap
